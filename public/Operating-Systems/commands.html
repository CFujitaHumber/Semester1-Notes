who am I?

The first thing someone new should do is start by learning what their username is.

If they already know, that's great, but outputting it to a terminal is a basic command users should know anyways.

to output your username type whoami into your terminal should look like this

username@hostname:~$ whoami
username

Who else is logged in?

If you want to know who is logged in at the same time you are use who.

Your terminal should look somewhat like the following:

username@hostname:~$ who
username seat0 2024-10-21 20:30 (login screen) username tty2 2024-10-21 20:30 (tty2)

Where are you?

Every terminal works from a specific directory called a present working directory. The pwd command outputs the PWD into terminal.

Your terminal should look somewhat like the following:

username@hostname:~$ pwd
/home/username

What is here?

If you ever need to know what files are in your PWD or any other directory, then you're gonna need to use ls. Short for 'list'; the ls command provides a list of everything in a directory.

Looks something like the following:

username@hostname:~$ ls
Desktop Documents Downloads Music Videos

To see every file including special and hidden files (files that start with '.' are hidden) you need the (-a) switch.

Use of the switch should similar to the following:

[username@hostname:~$ ls -a
. .. .cache .config Desktop Documents Downloads .local Music Videos

To see a long format you need the (-l) switch.

[username@hostname:~$ ls -a
drwxr-xr-x. 1 username username 0 Oct 26 20:30 Desktop
drwxr-xr-x. 1 username username 0 Oct 26 20:30 Documents
drwxr-xr-x. 1 username username 0 Oct 26 20:30 Downloads
drwxr-xr-x. 1 username username 0 Oct 26 20:30 Music
drwxr-xr-x. 1 username username 0 Oct 26 20:30 Videos

What does that mean?

reading long list format is easy once you know the format. It outputs:

  • File type and permission
  • number of links
  • owner of file
  • group associated to file
  • file size (bytes)
  • last modification date and time
  • name of file

how do you create files?

You can delete files through two commands: touch and mkdir.

Looks something like the following:

username@hostname:~$ touch example.filetype
username@hostname:~$ touch 'annoying file with spaces in it.filetype'
username@hostname:~$ mkdir example-directory
username@hostname:~$ mkdir 'annoying directory with spaces in it'

sometimes you want to create a directory and it's parents. Do the following with the parent switch (-p):

username@hostname:~$ mkdir -p example/example2

Maybe you want multiple files. Do the following with brace expansion or just type the extra files:

username@hostname:~$ touch example{1..10}
username@hostname:~$ touch example11 example12

how do you delete files?

You can delete files through two commands: rm and rmdir.

Looks something like the following:

username@hostname:~$ rm example.filetype
username@hostname:~$ rm 'annoying file with spaces in it.filetype'
username@hostname:~$ rmdir example-directory
username@hostname:~$ rmdir 'annoying directory with spaces in it'

rm command can delete files and directories, but in-order to delete directories you need to add the recursive switch (-r).

username@hostname:~$ rm -r example-directory
username@hostname:~$ rm -r 'annoying directory with spaces in it'

If the user wants to be clear and verify every file, they can use the verbose switch (-v) and the interactive switch (-i) with the recursive switch (-r). rm -vir will verify each action and the user must input a 'y' or 'n' (yes or no) respectively.

username@hostname:~$ rm -vir example
rm: decend into directory 'example'? y
rm: remove directory 'example2'? y
removed directory 'example/example2'
rm: remove directory 'example'? y
removed directory 'example'

rmdir removes empty directories. I'm not providing an example to this; maybe future updates.

How do you move files?

files are moved and renamed with mv; it stands for 'move'.

Looks something like the following:

username@hostname:~$ mv Documents/bee-movie-script.txt readme.txt

if you want to move to a specific directory use the target directory switch:

username@hostname:~$ mv -t Documents readme.txt

if you want to verify incase of overwrite use the interactive switch (-i):

username@hostname:~$ mv -i Documents/bee-movie-script.txt readme.txt
mv: overwrite 'readme.txt'? y

How do you copy files?

files are copied and renamed with cp; it stands for 'copy'. It also functions very similar to mv.

Looks something like the following:

username@hostname:~$ cp Documents/bee-movie-script.txt readme.txt

if you want to move to a specific directory use the target directory switch:

username@hostname:~$ cp -t Documents readme.txt

if you want to verify incase of overwrite use the interactive switch (-i):

username@hostname:~$ cp -i Documents/bee-movie-script.txt readme.txt
cp: overwrite 'readme.txt'? y

clear

the clear command clears the terminal of previous commands visually, but this doesn't mean the previous commands are removed from memory. It is very useful if you want to remove clutter from the terminal when working with multiple commands.

history

the history command is a command that outputs the entire bash history of commands.

pwd

the pwd outputs your PWD (Present working directory).

ls

the ls command shows the files in your PWD.

-a

-t
