Designing a Program

The process of designing a program can be summarised in following two steps

  1. Understand the task the program is to perform
  2. Determine the steps to be taken to perform the task

Description

Key Concepts:

  1. Input, Process, Output:
    • Programs receive input, process it (e.g., calculations), and provide output.
    • Input is stored in variables for further manipulation.
  2. Variables
    • Variables are storage locations in memory with specific names
    • Variables are declared with a name and a data type
    • Examples of valid variable names: grossPay, pay_rate, length
  3. Variable Declarations and Data Types

Output, Input and Variables

  • Data produces and displayed by the program is called output
  • Data received by the program is called Input.
  • When a program receives data, it stores it in a variable. A variable is a named storage location in memory.
  • Input is data that is typed on the keyboard. Once input is received, some process, such as a mathematical calculation, is usually performed on it. The results of the process are then sent out of the program as output.

Naming schemes

  • Variable names must be one words. They cannot contain spaces.
  • In most languages, punctuation characters cannot be used in variable names.
  • Use only alphabetic letters and numbers in variable names. The first character of a variable cannot be a number.

Data types

The data types that you are allowed to use depend on the programming language. For example, the Java language provides four data types for integer numbers, two data types for real numbers, one data type for strings and others.
We will use the following data types:

Integer
hold whole numbers, such as 42, 0, -99
Real
hold either whole numbers or numbers with fractional parts such as 3.5, 0, -3.0
String
hold any string of character, such as name, address, password, and so on.

Scope

a Scope is a contained section of a task-list.

Global Variable

Global Variables are a type of variable accessible to all task-lists for a solution; they are defined outside the scope of subtasks. Global variables are also called fields and are automatically initialised when the program executes.

  • A variable declared outside a module.
  • It is accessible to all the modules in an algorithm
  • It can be used by any module in the program; called the variable's scope.

Local Variable

Local Variable are a type of variable accessible to only the section of the task-list it is declared in; defined inside its scope. Local variables are not automatically initialised, and you should always assign them an initial value in the declaration statement.

  • A variable declared inside a module
  • It is accessible only to a particular module where it is declared.
  • It can only be used within the module's scope

Sequential Structure

Sequential Structure is a process of steps or events.
Scripting typically follows a structure of command in an order of tasks. See an example here.

Decision Structure

Decision Structure follows sequential-structure but changes the flow of sequence based on a condition.

Duel-Decision Structure

Duel-Decision Structure is a type of decision structure that changes the flow of sequence based on a condition's return.
In a Duel-Decision Structure the flow follows a condition. If the condition is true, then a separate sequence of tasks is followed; however, a false condition will result in a different sequence.

Top-Down Design

A complex problem may require sectioning into smaller tasks. This is part of Top-Down Design, where major tasks in a problem are recursively divided into subtasks.
Any programmer designing based on top-down principals needs to start at the topmost task and break those down into smaller subtasks; known as procedural programming. This is design can start through the creation of a hierarchy chart.

Hierarchy Chart

a hierarchy chart is a visualization of all tasks and their corrosponding subtasks through arrows and boxes.

[image]

Modules

After a hierarchy chart is made a programmer can break down the tasks into Modules. Modules are sections of an algorithm/process dedicated to a specific function/task. This allows larger programs to be systematically broken into small subtasks. This process of modules and Top-Down design allows for

  • Simpler code
  • Reusable code
  • Easier testing
  • Faster development
  • Easier understanding/readability of code

When a module is called by name, that module has been invoked directly.

Modularization

The process of dividing an algorithm into small functions is called modularization Modularization encapsulates variable data inside it's functions. This means that every module can contain their own variables local only to itself; no other module can access it. This is known as a Local Variable, where a variable is declared and accessible only to commands within a scope.
On the other hand, a Global Variable is a variable accessible to all functions/modules in an algorithm.

Functions

subtasks can turn only one local variable back to the parent tasks scope; called returning a value. This allows that previously local variable to be accessed by other subtasks. However, the parent task must catch the returning value through any sort of declaration or value reassignment otherwise it cannot be accessed.
When a module acts as a subtask that returns a value it is known as a function. Essentially a function is a module that returns a value.
