Computer Programs

Programming is the development of a solution to a problem. a computer program is a set of instructions that,when directed through ccomputer hardware, produces results. Design a solution to the program first, desk check this solution, and then code the program in a chosen programming language.

Designing a program

The process of designing a program can be summarized in following two steps:

  1. Understand the task that the program is to perform.
  2. Determine the steps that must be taken to perform the task.

Understand the task that the program is to perform

It is essential to understand what a program is supposed to do before you can determine the steps thhat the program will perform. The programmer studies the information that was gatherdf from the problem and creates al ist of different software requirements.

Determine the steps that must be taken to perform the task

after completing the first step, begin by breaking dowan the task into a series of steps. This is an example of an algorithm, which is a set of well - defined logical steps that must be taken to perform a task. for example, you want to design a program to add up a list of prices on a pocket calculator. to solve this problem we can make a task list.

  • Turn on calculator
  • Clear calculator
  • Key in dollar amount
  • Key in decimal point (.)
  • Key in cents amount
  • Press addition (+) key
  • Repeat the aboves four steps until all prices have been entered
  • Write down total price
  • Turn off calculator

Task list falls in the category of informal logic plan. In creating a task list, you can first solve the provlem by hand and then record the steps you took to solve the problem. We can create a formal logic plan by using techniques as:flowchart, psuedocode, and coputer program. In this course, we will focus on using flowchart and psudocode.

Desk Checking

Desk checking ensures our project is correct. We will produce somesample data and work throuhg the solution by hand to see if we get the right answer . It is important to complete this step to ensure that the answer is correct. For instance, if a car achieved 35 Kilomeres per litre, we would asssume that your program was incorrect, regardless of how you designed the program.
